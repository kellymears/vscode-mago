# Formatting

The extension registers as a VS Code formatting provider for PHP files, enabling format-on-save and manual formatting via the command palette.

## Format on Save

Add this to your VS Code settings:

```json
{
  "[php]": {
    "editor.defaultFormatter": "kellymears.vscode-mago",
    "editor.formatOnSave": true
  }
}
```

## Manual Formatting

- **Command palette**: `Mago: Format File`
- **Keyboard shortcut**: `Shift+Alt+F` (default VS Code format keybinding)
- **Right-click**: Format Document

## How It Works

1. The extension pipes your document text to `mago format --stdin-input` via stdin.
2. Mago formats the code according to your `mago.toml` configuration (presets, indent style, line width, etc.).
3. If the output differs from the original, a full-document text edit is applied.
4. If the output is identical, no edit is made (no unnecessary saves).

The formatting is **async** — VS Code's cancellation token is respected, so you can cancel a long-running format operation.

## Formatter Settings

The formatter itself is configured via `mago.toml`, not via VS Code settings. The extension passes `--workspace` to mago so it picks up your project config automatically.

Example `mago.toml` formatter configuration:

```toml
[format]
preset = "psr-12"
indent = "spaces"
indent-size = 4
line-width = 120
```

See the [mago documentation](https://github.com/carthage-software/mago) for all available formatter options.

## Error Handling

If formatting fails (e.g., syntax error in the file, mago crash), the original document is left unchanged. Errors are logged to the Mago output channel — set [`mago.trace.level`](/docs/configuration#magotracelevel) to `"error"` or higher to see them.
